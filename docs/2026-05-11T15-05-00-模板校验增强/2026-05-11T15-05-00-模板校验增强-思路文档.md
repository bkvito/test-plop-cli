# 模板校验增强思路文档

## 问题

- 现有自动校验只覆盖 `src/index.vue`
- 如果 `typing.ts`、`hook.ts`、`index.ts` 出错，当前脚本无法及时发现
- 报错信息偏通用，不利于快速定位具体出错文件

## 思考

- 若继续只校验 `index.vue`，模板验证价值有限
- 若直接做全文比对，模板稍微调整就容易误报，维护成本高
- 更合适的折中方式是“多文件 + 关键片段断言”

## 选择原因

- 关键片段能覆盖模板分流、变量替换、核心结构三类风险
- 文件维度的规则结构更容易维护
- 出错时直接报“模板组合 + 文件路径 + 缺少片段”，排查成本更低
