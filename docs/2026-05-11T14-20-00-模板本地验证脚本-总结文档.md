# 模板本地验证脚本总结

## 本次完成内容

- 在 `package.json` 中补充了模板本地验证脚本，方便修改 `.hbs` 模板后直接执行命令验证生成结果。
- 新增了 `general/setup`、`general/normal`、`single-pick/setup`、`multiple-pick/setup` 四类验证命令。
- 新增了一条 `verify:template:all` 聚合命令，用于一次性验证常用模板输出。

## 验证命令示意

```bash
npm run verify:template:general:setup
npm run verify:template:all
```

## 验证思路

- 所有验证命令都直接调用 `bin/cli.mjs`
- 通过显式传入 `templateType` 与 `vueTemplateType`，避免交互模式下误选
- 统一输出到 `tmp-output` 目录，并使用 `overwrite` 覆盖旧结果，便于反复验证

## 当前不足

- 当前验证仍以“检查生成文件内容”为主，还没有自动比对输出文件内容是否符合预期
- 还没有覆盖 `single-pick/normal` 与 `multiple-pick/normal` 场景
- 还没有自动清理 `tmp-output` 中历史生成目录

## 后续可选优化

- 增加更多脚本，覆盖全部模板组合
- 增加一个简单的校验脚本，对生成的 `index.vue` 做关键片段断言
- 将验证步骤写入 README，降低后续使用成本
