# 模板验证优化思路文档

## 问题背景

- 当前已经有部分模板验证命令，但覆盖范围还不完整
- 现有验证方式主要依赖人工打开生成文件检查，效率较低
- 交互式命令容易因为误选模板类型导致判断偏差

## 思考过程

- 若只继续往 `package.json` 里追加命令，虽然能补齐覆盖范围，但仍然缺少自动校验能力
- 若把“生成 + 断言”都塞进一行脚本字符串里，可读性和维护性都会比较差
- 更稳妥的方式是增加一个独立脚本文件，让 `package.json` 只做入口映射

## 方案取舍

- 选择“脚本文件 + npm scripts”组合
- 不引入测试框架，保持实现轻量
- 断言只检查关键片段，不做过重的全文比对，避免模板小调整时频繁误报
