# 模板验证优化总结

## 本次完成内容

- 补齐了 `single-pick/normal` 与 `multiple-pick/normal` 的本地验证命令
- 新增 `scripts/verify-templates.mjs`，支持统一生成并自动校验关键片段
- 更新 README，补充模板改动后的推荐验证流程
- 将验证产物收敛到 `tmp-output/template-verify`，避免误删仓库原有示例目录

## 产出效果

- 修改模板后可以直接执行单项验证命令快速检查
- 执行 `npm run verify:template:all` 时，会连同关键片段断言一起完成
- 模板验证入口更加固定，降低了交互误操作的概率

## 当前不足

- 自动校验目前只覆盖 `src/index.vue`，还没有覆盖 `typing.ts`、`hook.ts` 等文件
- 关键片段断言仍然偏轻量，适合发现明显问题，但不能替代更细粒度的内容检查

## 后续可继续优化方向

- 将 `typing.ts`、`hook.ts`、`data.ts` 也纳入自动校验
- 为不同模板类型设计更有针对性的断言片段，而不只校验通用结构
- 视需要再引入更正式的测试体系
