# 模板验证优化实施方案

## 目标

- 补齐全部模板组合的本地验证命令
- 增加自动校验脚本，减少人工逐个打开文件检查的成本
- 将模板验证流程写入 README，降低后续维护门槛

## 实施思路

1. 在 `package.json` 中为六种模板组合分别提供脚本命令
2. 新增一个独立的校验脚本，统一执行生成、读取结果、断言关键片段
3. 在 README 中增加“模板改动后的本地验证”说明

## 伪代码示意

```js
for each 模板组合:
  先生成测试组件
  再读取 tmp-output 下的 src/index.vue
  检查组件名、Vue 风格标识、hook 调用片段
  任何一步失败则直接报错退出
```

## 风险控制

- 不修改业务模板内容，只补充验证能力
- 测试输出固定写入 `tmp-output/template-verify`，避免影响仓库中已有的示例输出目录
- 校验逻辑集中在单独脚本中，避免把复杂判断堆进 `package.json`
