# 三套业务模板重构总结文档

## 1. 本轮已完成内容

本轮工作完成了设计方案修订，已经把需求口径从“单维模板类型”修正为“双维模板体系”。

本次更新后的核心定义如下：

1. `templateType`
   业务模板类型：
   - `general`
   - `single-pick`
   - `multiple-pick`

2. `setup/normal`
   Vue 语法风格，只影响 `index.vue`

## 2. 本轮修订点

相较于上一版方案，本轮主要修正了以下内容：

1. 修正了参数语义
   不再把 `setup/normal` 误当成业务模板类型。

2. 修正了目录设计
   顶层目录按业务模板划分，每个业务模板下再区分 `index.setup.vue.hbs` 与 `index.normal.vue.hbs`。

3. 修正了生成逻辑
   变为“业务模板目录选择 + Vue 文件风格选择”的二段式生成逻辑。

4. 修正了验证范围
   后续闭环验证不再是 3 种情况，而是 6 种组合情况。

5. 明确了输出命名规则
   `index.setup.vue.hbs` 与 `index.normal.vue.hbs` 只是模板源文件名，最终输出文件名始终是 `index.vue`。

## 3. 当前版本结论

当前设计已经能够准确表达你的业务意图，后续可以直接据此进入实施阶段。

本次修订后的关键结论是：

```txt
业务模板类型决定整套模板语义，
Vue 风格只决定 index.vue 的写法。
```

并且：

```txt
setup / normal 只影响模板选择，
不影响最终输出文件名。
```
