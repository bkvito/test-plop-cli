# Vitest 与 CLI E2E 测试方案实施方案

## 目标

- 引入 Vitest 作为正式测试框架
- 用集成测试覆盖 `generateComponent()` 的生成契约
- 用 CLI E2E 测试覆盖真实命令入口
- 降低模板内容微调时同步修改测试断言的频率

## 实施步骤

1. 在 `package.json` 中增加 `test` 与 `test:watch` 脚本
2. 增加 Vitest 开发依赖
3. 新增 `tests/generate.test.mjs`，覆盖模板组合、文件生成、变量替换、冲突策略和非法参数
4. 新增 `tests/cli.e2e.test.mjs`，覆盖版本输出、非交互生成和严格非交互失败路径
5. 执行 `npm run test` 做闭环验证

## 伪代码

```txt
生成函数测试:
  遍历模板组合
  调用 generateComponent
  检查文件存在、变量替换完成、入口导出正确

CLI E2E 测试:
  使用 node bin/cli.mjs 执行真实命令
  检查退出码、输出文本和生成文件
```

## 风险控制

- 测试产物写入系统临时目录，避免污染仓库
- 测试只验证生成契约，不绑定大量模板正文
- 保留现有 `verify:template:*` 命令，避免一次性切换过猛
