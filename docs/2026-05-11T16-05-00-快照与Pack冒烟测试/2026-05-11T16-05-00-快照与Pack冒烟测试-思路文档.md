# 快照与 Pack 冒烟测试思路文档

## 背景

已有的 Vitest 测试主要验证生成契约和 CLI 入口，但还没有记录完整模板输出。对于模板类项目，完整输出变化本身就是重要信号。

## 思考

- 快照测试适合模板稳定阶段，用来发现完整文件输出变化
- 冒烟测试适合发布前，用来确认 npm 包至少包含运行所需文件
- `npm pack` 后直接安装会触发依赖解析，可能引入网络不稳定因素，因此本阶段先检查包结构
- npm 默认 cache 可能受本机权限影响，因此测试中使用临时 cache 隔离

## 取舍

- 选择 Vitest snapshot，而不是维护独立 golden file 目录
- Pack 冒烟测试不执行 `npm install <tgz>`，避免测试变慢和依赖网络
- 保持现有生成函数测试和 CLI E2E 测试不变，快照与冒烟测试作为补充层
