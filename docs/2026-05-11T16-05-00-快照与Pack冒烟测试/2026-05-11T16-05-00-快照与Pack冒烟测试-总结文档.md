# 快照与 Pack 冒烟测试总结

## 本次完成内容

- 新增模板完整输出快照测试
- 新增 `npm pack` 冒烟测试
- 将模板输出变化和发布包关键文件完整性纳入正式测试流程
- 新增 `test:update-snapshot` 与 `test:pack` 命令，方便单独执行快照更新和 pack 冒烟测试

## 当前收益

- 模板输出发生完整文件级变化时，测试会明确提示快照差异
- 发布前可以确认 tarball 能生成，且包含 CLI、生成逻辑和模板目录
- 测试产物都写入临时目录，不污染仓库运行目录
- Pack 冒烟测试使用临时 npm cache，避免依赖用户全局 npm cache 权限
- 已执行 `npm run test`，4 个测试文件、19 个用例全部通过

## 当前不足

- Pack 冒烟测试当前只检查包结构，还没有安装 tarball 后执行 CLI
- 快照测试会在模板大量调整时产生较多 diff，需要人工确认后更新

## 后续方向

- 如需更接近真实用户环境，可补充离线安装 tarball 后运行 CLI 的测试
- 如发布包内容需要瘦身，可继续增加不应进入包内的文件断言
