# 快照与 Pack 冒烟测试实施方案

## 目标

- 增加模板完整输出的快照测试
- 增加 `npm pack` 冒烟测试
- 让发布前测试覆盖“生成结果变化”和“发布包关键文件完整性”

## 实施步骤

1. 新增 `tests/template.snapshot.test.mjs`
2. 生成六种模板组合，并对完整输出文件做快照断言
3. 新增 `tests/package-smoke.test.mjs`
4. 执行 `npm pack --json --pack-destination`，检查 tarball 和关键运行文件
5. 执行 `npm run test`，生成并验证快照

## 伪代码

```txt
快照测试:
  for each 模板组合:
    生成组件
    读取全部输出文件
    和 snapshot 比对

Pack 冒烟测试:
  执行 npm pack
  检查 tgz 文件存在
  检查 package.json、bin、src、plop-templates 等关键文件进入包内
```

## 风险控制

- 快照测试使用系统临时目录，不污染仓库生成目录
- Pack 冒烟测试使用临时 npm cache，只检查包结构，不执行安装，避免引入网络依赖
- 快照内容变化需要显式通过 Vitest 更新，适合模板稳定后的回归保护
